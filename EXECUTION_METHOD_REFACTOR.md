# إعادة هيكلة نظام إدارة المنتجات - نقل الحقول لأنواع الطلب

## نظرة عامة
تم إعادة هيكلة نظام إدارة المنتجات بالكامل لنقل **جميع ميزات الحقول المخصصة والكميات/الفئات** من مستوى "النوع (Region)" إلى مستوى "نوع الطلب (Execution Method)" حصرياً. هذا يجعل كل طريقة تنفيذ وحدة مستقلة تماماً بكل بياناتها وميزاتها.

## التغييرات الرئيسية

### 1. حذف الحقول من مستوى النوع (Region)
تم حذف الحقول التالية من مستوى النوع:
- ❌ الحقل المخصص (customInput) على مستوى النوع
- ❌ الكميات/الفئات (denominations) على مستوى النوع
- ❌ جميع الميزات المرتبطة بها (التفعيل، التعديل، الحذف، إلخ)

### 2. نقل جميع الميزات إلى أنواع الطلب
تم نقل **جميع الميزات** إلى مستوى "نوع الطلب":

#### أ. الحقل المخصص (Custom Input)
لكل طريقة تنفيذ يمكن الآن:
- ✅ تفعيل/تعطيل حقل مخصص
- ✅ تعيين عنوان الحقل (مثال: "Player ID", "Account Link")
- ✅ تعيين نص توضيحي (Placeholder)
- ✅ تحديد ما إذا كان الحقل مطلوباً (إجباري)

#### ب. الكميات/الفئات (Denominations)
لكل طريقة تنفيذ يمكن الآن:
- ✅ إضافة كميات/فئات مختلفة
- ✅ تعيين سعر لكل كمية/فئة
- ✅ **تفعيل/تعطيل التوفر** (Enable/Disable Availability) لكل كمية
- ✅ حذف الكميات غير المطلوبة

### 3. هيكل البيانات الجديد

```
منتج (Product)
├── نوع 1 (Region)
│   ├── نوع الطلب 1
│   │   ├── customInput: { enabled, label, placeholder, required }
│   │   ├── denominations: [
│   │   │   { id, label, price, isAvailable },
│   │   │   { id, label, price, isAvailable }
│   │   │ ]
│   │   └── apiConfig: { type, serviceId, providerName }
│   │
│   └── نوع الطلب 2
│       ├── customInput: { enabled, label, placeholder, required }
│       ├── denominations: [...]
│       └── apiConfig: {...}
│
└── نوع 2 (Region)
    └── ...
```

## الفوائد

### 1. **مرونة أكبر**
كل طريقة تنفيذ لديها بيانات مختلفة تماماً بدون تأثر بالطرق الأخرى.

### 2. **وضوح أفضل**
جميع الخيارات الخاصة بطريقة تنفيذ معينة موجودة في مكان واحد.

### 3. **إدارة أسهل**
لا حاجة للتنقل بين عدة أقسام - كل شيء في بطاقة واحدة.

### 4. **استقلالية كاملة**
كل طريقة تنفيذ تعمل بشكل مستقل تماماً دون تأثر بالطرق الأخرى.

## خطوات الاستخدام

### للمسؤولين:
1. الذهاب إلى لوحة التحكم
2. الذهاب إلى قسم "المنتجات"
3. تحرير منتج موجود أو إنشاء منتج جديد
4. الذهاب إلى تبويب "الخيارات"
5. تحديد نوع المنتج
6. توسيع قسم "أنواع الطلب"
7. **لكل طريقة تنفيذ:**
   - تفعيل الحقل المخصص وتخصيصه (اختياري)
   - إضافة الكميات/الفئات المناسبة
   - تفعيل/تعطيل التوفر لكل كمية
   - حفظ المنتج

### للمطورين:
- استخدام `orderTypeObj?.customInput` للحقل المخصص
- استخدام `orderTypeObj?.denominations` للكميات
- الأولوية الجديدة: Execution Method > Region > Product
- كل طريقة تنفيذ مستقلة تماماً

## ملاحظات تقنية

- **التوافقية العكسية**: البيانات القديمة ستستمر في العمل
- **قاعدة البيانات**: لا تغييرات في بنية قاعدة البيانات
- **API الخلفية**: لا تغييرات في الـ Backend
- **الواجهة الأمامية**: جميع التعديلات في Frontend فقط

## الفروقات من الإصدار السابق

| الميزة | الإصدار السابق | الإصدار الجديد |
