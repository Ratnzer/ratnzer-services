# نظام التخصيص متعدد المستويات - الحقول والكميات

## نظرة عامة
تم تطوير نظام تخصيص متقدم يسمح بإدارة **الحقول المخصصة (Custom Input)** و **الكميات/الفئات (Denominations)** في **ثلاثة مستويات مختلفة** مع نظام أولويات ذكي يحدد أي البيانات يتم استخدامها.

## المستويات الثلاثة

### 1️⃣ المستوى الأول: المنتج العام (Product Level)
**الموقع في لوحة التحكم**: تبويب "التفاصيل" → قسم "الكميات / الفئات"

**الميزات**:
- ✅ إضافة كميات/فئات عامة للمنتج
- ✅ تعديل وحذف الكميات
- ✅ هذه تكون **قيم افتراضية** لجميع الأنواع وأنواع الطلبات

**الاستخدام**:
- عندما تريد كميات موحدة لجميع الأنواع والطلبات
- مثال: منتج بسيط بكمية واحدة فقط

---

### 2️⃣ المستوى الثاني: نوع المنتج (Region Level)
**الموقع في لوحة التحكم**: تبويب "الخيارات" → قسم "أنواع الطلب" → "الحقل المخصص والكميات لهذا النوع"

**الميزات**:
- ✅ تفعيل/تعطيل **حقل مخصص** لهذا النوع (مثل: Player ID)
- ✅ تخصيص عنوان الحقل والنص التوضيحي
- ✅ تحديد ما إذا كان الحقل مطلوباً (إجباري)
- ✅ إضافة كميات/فئات خاصة بهذا النوع
- ✅ تفعيل/تعطيل التوفر لكل كمية

**الاستخدام**:
- عندما تريد كميات مختلفة لكل نوع منتج
- عندما تحتاج حقل مخصص لنوع معين فقط
- مثال: نوع "عالمي" يحتاج Player ID، ونوع "محلي" لا يحتاجه

---

### 3️⃣ المستوى الثالث: نوع الطلب (Execution Method Level)
**الموقع في لوحة التحكم**: تبويب "الخيارات" → قسم "أنواع الطلب" → داخل كل نوع طلب

**الميزات**:
- ✅ تفعيل/تعطيل **حقل مخصص** لهذا النوع من الطلب فقط
- ✅ تخصيص عنوان الحقل والنص التوضيحي
- ✅ تحديد ما إذا كان الحقل مطلوباً (إجباري)
- ✅ إضافة كميات/فئات خاصة بهذا النوع من الطلب
- ✅ تفعيل/تعطيل التوفر لكل كمية
- ✅ ربط مباشر مع إعدادات التسليم التلقائي

**الاستخدام**:
- عندما تريد كميات مختلفة لكل نوع طلب
- عندما تحتاج حقل مخصص لنوع طلب معين فقط
- مثال: نوع طلب "شحن مباشر" يحتاج Player ID، ونوع طلب "إرسال عبر Email" يحتاج Email

## نظام الأولويات (Priority System)

عند عرض المنتج للمستخدم، يتم تطبيق النظام التالي:

```
┌─────────────────────────────────────────────┐
│  هل توجد بيانات في نوع الطلب؟              │
│  (Execution Method Level)                   │
└────────────┬────────────────────────────────┘
             │
        نعم │ لا
             │
    ┌────────▼─────────┐
    │ استخدم بيانات    │
    │ نوع الطلب        │
    │ (الأولوية الأولى)│
    └────────┬─────────┘
             │
        ┌────▼──────────────────────────┐
        │ هل توجد بيانات في النوع؟     │
        │ (Region Level)                │
        └────┬───────────────────────────┘
             │
        نعم │ لا
             │
    ┌────────▼──────────┐
    │ استخدم بيانات     │
    │ النوع             │
    │ (الأولوية الثانية)│
    └────────┬──────────┘
             │
        ┌────▼──────────────────────────┐
        │ هل توجد بيانات في المنتج؟    │
        │ (Product Level)               │
        └────┬───────────────────────────┘
             │
        نعم │ لا
             │
    ┌────────▼──────────┐
    │ استخدم بيانات     │
    │ المنتج            │
    │ (الأولوية الثالثة)│
    └────────┬──────────┘
             │
        ┌────▼──────────────────────────┐
        │ لا توجد بيانات               │
        │ (لا تعرض أي كميات)           │
        └─────────────────────────────────┘
```

## أمثلة عملية

### مثال 1: منتج شحن الألعاب (متعدد الأنواع والطلبات)

```

│
├─ المستوى 1 (المنتج العام)
│  └─ كميات افتراضية:
│     ├─ 50 UC - $0.99
│     ├─ 100 UC - $1.99
│     └─ 500 UC - $9.99
│
├─ نوع 1: عالمي (Global)
│  ├─ حقل مخصص: "Player ID" (مطلوب)
│  ├─ كميات خاصة:
│  │  ├─ 60 UC - $0.99 ✅
│  │  ├─ 300 UC - $4.99 ✅
│  │  └─ 600 UC - $9.99 ❌ (معطل)
│  │
│  ├─ نوع الطلب 1: شحن مباشر
│  │  ├─ حقل مخصص: "Game Account Email" (مطلوب)
│  │  └─ كميات خاصة:
│  │     ├─ 100 UC - $1.49 ✅
│  │     └─ 500 UC - $6.99 ✅
│  │
│  └─ نوع الطلب 2: شحن عبر API
│     ├─ حقل مخصص: (معطل)
│     └─ كميات خاصة:
│        ├─ 60 UC - $0.89 ✅
│        └─ 300 UC - $4.49 ✅
│
└─ نوع 2: محلي (Local)
   ├─ حقل مخصص: (معطل)
   ├─ كميات خاصة:
   │  ├─ 50 UC - $0.79 ✅
   │  └─ 250 UC - $3.99 ✅
   │
   └─ نوع الطلب 1: شحن محلي
      ├─ حقل مخصص: "رقم الهاتف" (مطلوب)
      └─ كميات خاصة:
         ├─ 50 UC - $0.99 ✅
         └─ 250 UC - $4.99 ✅
```

**كيفية الاستخدام**:
1. عندما يختار المستخدم "عالمي" → يرى كميات النوع (60, 300, 600 UC)
2. عندما يختار "شحن مباشر" → يرى كميات نوع الطلب (100, 500 UC) وحقل "Game Account Email"
3. عندما يختار "محلي" → يرى كميات النوع (50, 250 UC) ولا يرى حقل مخصص
4. عندما يختار "شحن محلي" → يرى كميات نوع الطلب (50, 250 UC) وحقل "رقم الهاتف"

## الفوائد

### 1. **مرونة عالية جداً**
- يمكنك تحديد كميات مختلفة لكل مستوى
- يمكنك تفعيل/تعطيل حقول مخصصة حسب الحاجة

### 2. **سهولة الإدارة**
- إذا أردت تغيير كمية واحدة لجميع الأنواع، غيرها في المستوى الأول
- إذا أردت تغيير كمية لنوع معين، غيرها في المستوى الثاني

### 3. **تجنب التكرار**
- لا تحتاج لإضافة نفس الكميات في كل مكان
- استخدم الأولويات للتحكم في ما يظهر

### 4. **تجربة مستخدم أفضل**
- المستخدم يرى فقط الخيارات المناسبة لاختياره
- الحقول المخصصة تظهر فقط عند الحاجة

## خطوات الاستخدام

### للمسؤولين:

#### الخطوة 1: إضافة كميات عامة (اختياري)
1. الذهاب إلى لوحة التحكم → المنتجات
2. تحرير منتج
3. تبويب "التفاصيل"
4. قسم "الكميات / الفئات (المستوى العام للمنتج)"
5. إضافة الكميات الافتراضية

#### الخطوة 2: تخصيص حسب النوع (اختياري)
1. تبويب "الخيارات"
2. اختيار نوع المنتج
3. توسيع قسم "أنواع الطلب"
4. تفعيل الحقل المخصص (إذا أردت)
5. إضافة كميات خاصة بهذا النوع

#### الخطوة 3: تخصيص حسب نوع الطلب (اختياري)
1. نفس القسم السابق
2. توسيع بطاقة نوع الطلب
3. تفعيل الحقل المخصص (إذا أردت)
4. إضافة كميات خاصة بهذا النوع من الطلب

### للمطورين:

```javascript
// الحصول على البيانات بناءً على الأولويات
const getCustomInput = (product, region, orderType) => {
  // الأولوية 1: نوع الطلب
  if (orderType?.customInput?.enabled) {
    return orderType.customInput;
  }
  
  // الأولوية 2: النوع
  if (region?.customInput?.enabled) {
    return region.customInput;
  }
  
  // الأولوية 3: المنتج
  return null; // لا يوجد حقل مخصص
};

const getDenominations = (product, region, orderType) => {
  // الأولوية 1: نوع الطلب
  if (orderType?.denominations?.length > 0) {
    return orderType.denominations;
  }
  
  // الأولوية 2: النوع
  if (region?.denominations?.length > 0) {
    return region.denominations;
  }
  
  // الأولوية 3: المنتج
  if (product?.denominations?.length > 0) {
    return product.denominations;
  }
  
  return []; // لا توجد كميات
};
```

## ملاحظات تقنية

### التوافقية العكسية
- البيانات القديمة ستستمر في العمل بدون تعديلات
- النظام الجديد يعمل بجانب النظام القديم

### قاعدة البيانات
- لا تغييرات في بنية قاعدة البيانات
- جميع البيانات تُخزن في نفس الأماكن السابقة

### الأداء
- لا تأثير على الأداء
- النظام يستخدم نفس آليات التخزين والاسترجاع
