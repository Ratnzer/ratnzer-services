# ربط الحقول المخصصة والكميات بطرق التنفيذ

## نظرة عامة
تم تحديث نظام إدارة المنتجات ليسمح بتخصيص الحقول والكميات/الفئات لكل **طريقة تنفيذ (Execution Method)** بشكل مستقل. هذا يوفر مرونة أكبر في إدارة المنتجات المختلفة.

## التغييرات الرئيسية

### 1. تحديث واجهة البيانات (`src/types.ts`)

تم إضافة حقلين جديدين إلى واجهة `ExecutionMethod`:

```typescript
export interface ExecutionMethod {
  id: string;
  name: string;
  isAvailable: boolean;
  apiConfig?: ApiConfig;
  denominations?: Denomination[]; // الكميات/الفئات الخاصة بهذه الطريقة
  customInput?: CustomInputConfig; // الحقل المخصص الخاص بهذه الطريقة
}
```

### 2. تحديث لوحة التحكم (`src/pages/Admin.tsx`)

#### أ. إضافة واجهة لتخصيص الحقول المخصصة
لكل طريقة تنفيذ يمكن الآن:
- تفعيل/تعطيل حقل مخصص
- تعيين عنوان الحقل (مثال: "Player ID", "Account Link")
- تعيين نص توضيحي (Placeholder)
- تحديد ما إذا كان الحقل مطلوباً (إجباري)

#### ب. إضافة واجهة لإدارة الكميات/الفئات
لكل طريقة تنفيذ يمكن الآن:
- إضافة كميات/فئات مختلفة
- تعيين سعر لكل كمية/فئة
- حذف الكميات غير المطلوبة

### 3. تحديث عرض المنتج (`src/components/ProductDetailsModal.tsx`)

تم تحديث منطق الأولويات لاختيار الحقل المخصص والكميات:

**الأولوية (من الأعلى إلى الأقل):**
1. **طريقة التنفيذ (Execution Method)** - الأولوية الأعلى
2. **نوع المنتج (Region)** - الأولوية الوسطى
3. **المنتج (Product)** - الأولوية الأقل

هذا يعني أن إذا كانت طريقة التنفيذ لديها حقل مخصص، سيتم استخدامه بدلاً من حقل النوع أو المنتج.

## الفوائد

1. **مرونة أكبر**: كل طريقة تنفيذ يمكن أن تحتوي على بيانات مختلفة تماماً
2. **تجربة مستخدم أفضل**: يتم عرض الحقول والكميات المناسبة فقط
3. **سهولة الإدارة**: واجهة واضحة لتخصيص كل طريقة تنفيذ
4. **توافقية عكسية**: الأنظمة القديمة ستستمر في العمل بدون تغييرات

## الملفات المعدلة

   - إضافة `customInput` و `denominations` إلى `ExecutionMethod`

   - دوال للتعامل مع حفظ وتحديث البيانات

3. **src/components/ProductDetailsModal.tsx**
   - تحديث منطق الأولويات لاختيار الحقل المخصص
   - تحديث منطق الأولويات لاختيار الكميات

## الخطوات التالية

### للمسؤولين:
1. الذهاب إلى لوحة التحكم
2. الذهاب إلى قسم "المنتجات"
3. تحرير منتج موجود أو إنشاء منتج جديد
4. الذهاب إلى تبويب "الخيارات"
5. تحديد نوع المنتج
6. توسيع قسم "طرق التنفيذ"
7. لكل طريقة تنفيذ:
   - تفعيل الحقل المخصص وتخصيصه
   - إضافة الكميات/الفئات المناسبة

### للمطورين:
- يمكن الآن الاستفادة من `executionMethodObj?.customInput` و `executionMethodObj?.denominations` في الكود
- الأولوية الجديدة: Execution Method > Region > Product

## ملاحظات تقنية

- البيانات محفوظة في قاعدة البيانات بنفس الطريقة السابقة
- لا توجد تغييرات في API الخلفية (Backend)
- جميع التعديلات في الواجهة الأمامية (Frontend) فقط
- التوافقية العكسية مضمونة

## الدعم والمساعدة

في حالة وجود أي مشاكل أو استفسارات، يرجى التواصل مع فريق التطوير.
